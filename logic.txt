The application logic centers on a 4-step workflow to generate a weekly newsletter:

### 1. Article Ingestion & Sourcing
- Import: Users upload an Excel sheet ( .xlsx ) containing article links, titles, and metadata. The system parses this file, mapping various column headers (e.g., "Topic" -> "title") to a standardized internal format.
- AI Generation: Alternatively, users can generate articles using an AI prompt (Anthropic Opus 4.6 or another). The system enforces a strict JSON schema for the AI response to ensure valid article objects are created.
- Data Structure: Articles are stored with a status ( N , Y , YM , M, COOL FINDS ) and categorization ( MED , THC , CBD , INV ).
The number by the article dictates what number it is from top to bottom. IF there are two columns then even numbers will be on the right.
The articles that are chosen go to the next step where images are found for them in the Image Selection Page
### 2. Image Selection 
- Search Logic: Users search for icons/images for each article. The system validates search terms (1-2 words) and status eligibility (must be Y , YM , or COOL FINDS ).
Users have the option of uploading their own image
- Smart Querying: The backend ( freepik.js ) intelligently adjusts search terms based on context:
  - Since no two images can be the same there are multiple images to choose from and synonyms are encouraged
  - People: If the icon represents a person then it try to find a kawaii type image. If the api doesnt allow one to specify pack then kawaii can be one of the two keywords looking for the person.
The image that flaticon downloads should be 128 by 128
- Filtering: It prioritizes "flat" and "fill" styles, penalizes "outline" or "black & white" icons, and specifically boosts "kawaii" styles if requested.
All Images except for the inspirational photo are automatically resized with python to be 90 by 90 before being uploaded to the server
### 3. Content & Templating
- Text: Users provide Customized Newsletter, Same valediction will be used on all newsletters.
User can pick the valediction that he wants
- Templates: Users can upload custom HTML templates for different sections ( Master , MED , THC , CBD , INV ).
- Preview: The system generates HTML previews by injecting article data and images into these templates.
### 4. Publishing & Sync
- Manifest Generation: A manifest file is created listing all articles and their associated images.
- FTP Upload: The system connects to a GoDaddy FTP server to:
  1. Upload selected images to a week-specific folder (e.g., 2023-10-27/images/ )
	These images are then inserted into the newsletter template.
  2. Upload the final newsletter HTML.
Database retains all data including a json of the final newsletter
- Public URL Linking: After upload, the local database is updated with the public URLs of the uploaded images, replacing local preview paths.
- Cool Finds Limit: The system automatically enforces a limit of 2 "Cool Find" articles per category during the export process.

### Core Tech Stack
- Backend: Express.js with a file-based JSON database (supports Supabase switch).
- Frontend: RVanilla JS with view-based rendering (recently refactored to static HTML + hydration).
- Services: xlsx for sheets, basic-ftp for deployment, and openai / anthropic for AI.
